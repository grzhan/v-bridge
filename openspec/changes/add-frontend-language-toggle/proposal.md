## 摘要
为前端加入轻量级国际化层，让用户可以在登录页及登录后的应用头部在简体中文与英文之间切换。语言选择必须保存在 `localStorage` 中，这样界面能够以用户上次的语言加载，同时不影响任何后端契约。

## 背景
- 登录体验（`frontend/src/features/auth/pages.tsx`）和登录后的壳层（`frontend/src/components/layout/app-shell.tsx`）目前都写死为中文，英文用户无法舒适使用界面。
- 目前没有记录本地化需求的规范或能力，前端也没有共享的语言状态，因此切换界面语言只能通过刷新或手动修改。
- 后端不受影响。本次变更完全发生在前端：暴露语言切换、翻译可见文案，并在浏览器中缓存用户偏好。

## 目标
- 引入一个仅存在于前端的本地化机制，支持 zh-CN 与 en 的翻译。
- 在登录页和登录后的头部新增语言切换按钮，允许用户在登录前后都能切换语言。
- 确保语言选择会写入并从 `localStorage` 读取，且在没有记录时默认中文。
- 首次翻译范围限定在登录表单（标题、标签、按钮、辅助文字、可控的错误信息）和头部（角色文案、退出按钮文案、切换提示）。其他区域留待未来扩展。

## 非目标
- 翻译登录和头部以外的其他页面。
- 修改后端 API 或认证行为。
- 引入服务端驱动的语言协商。

## 影响
- 体验：英文用户可以登录并理解仪表盘的高层语义。
- 技术：新增共享的语言上下文/Provider 与简单的翻译表；使用 `localStorage` 持久化用户偏好。
- 风险：低。需要对登录、退出和导航做回归验证，确保语言状态在路由切换后仍然有效。

## 待确认问题
- 无；请求中已给定默认语言（中文）与存储机制（`localStorage`）。
